# 14.5 PageSpeed/Lighthouse Insights

A big factor of SEO today is how your Web site performs. A site that contains excellent information that is in high demand can still rank poorly if it loads slowly, is hard to render on some devices, or isn't mobile-friendly. We've already taken care of mobile-friendliness when we discussed _responsive design_, but just because our site is formatted for mobile devices doesn't mean that it will perform well. In fact, I'm sure you've noticed that pages load slowly. That's what we're going to be discussing in this tutorial. There are several tools available to help us analyze and improve our site's performance, and we're going to discuss two of them: _Google PageSpeed Insights_ and _Lighthouse_. Both of these tools are provided by Google and are based on the _Lighthouse_ engine. _PageSpeed Insights_ are available within the _Site Kit_ plugin and are also available via a [public Web interface](https://pagespeed.web.dev/). They are based on both real-world data and simulated data, but they require that the site you are analyzing is publicly accessible. _Lighthouse_ is built into the developer tools of Chromium-based browsers such as Google Chrome and Microsoft Edge, so you can use it to analyze any site, including your local development site.

For future reference, you can access _PageSpeed Insights_ from within the _Site Kit_ plugin by going to _Site Kit > Speed_ on the _Admin menu_.

Let's walk through how to use _Lighthouse_ to analyze our local development site and this will help you to understand the information that is available in both tools.

1. Open your site, preferably in Google's Chrome's Incognito mode or Edge's InPrivate mode, and press Ctrl-Shift-I to open the developer tools.
2. Click the _Lighthouse_ tab. If you don't see it, click the double right arrow in Chrome or the plus sign in Edge to see more tabs and then click _Lighthouse_.
3. You can choose to analyze the performance of the page for mobile devices desktop devices. You can also choose to include or exclude various categories of information, such as _Performance_, _Accessibility_, _Best Practices_, and _SEO_. You can also specify the Mode. You can read the [documentation from Google for more information about the mode choices](https://github.com/GoogleChrome/lighthouse/blob/HEAD/docs/user-flows.md). For now, let's use the _Navigation_ mode to analyze all categories on mobile devices by keeping the default settings.
4. Click _Analyze page load_ and wait for the results to load, which will take a moment.
5. You will see a score for each category as well as various pieces of information and recommendations for improving the performance of your site in each category. You can click on each category to see more details about the information and recommendations for that category.
6. For example, one recommendation is to improve image delivery. You can click on the recommendation to see more information about it, including the specific images that are causing the problem and suggestions for how to fix it. There are plugins that can help us with this and we're going to install a couple of them.
7. Keep the Lighthouse report open and navigate to the WordPress Admin Area in another browser tab.
8. Find _Plugins_ on the _Admin menu_ and click _Add New Plugin_.
9. Search for _Performance Lab_ and install and activate the plugin by the _WordPress Performance Team_.
10. Click _Settings_ to load the _Settings_ page with the new plugin active and then click _Performance_ under _Settings_.
11. _Performance Lab_ includes features that improve performance and that will likely become part of _WordPress Core_ eventually. They are being shared with the community for further testing. There are several features here that might help us. Let's start by activating _Enhanced Responsive Images_, which you can read more about by clicking its name.
12. Return to the Lighthouse report and click the _Reports_ dropdown, which contains a time and your site URL.
13. Click _New Report_.
14. Leave the settings as they are and click _Analyze Page Load_ again.
15. Lighthouse will run the analysis again and you can click on _Performance_ to see the updated information. You'll hopefully see the score increase and the recommendations shift since you have begun to address some of the issues, but the _improve image delivery_ recommendation will likely still be there since we haven't fully addressed that issue yet. The _Enhanced Responsive Images_ feature of _Performance Lab_ helps with this issue by adding the necessary attributes to your images so that browsers can choose the best image to load based on the device and screen size, but it doesn't actually create new images in different sizes or formats, which is what is needed to fully address this issue. There are other plugins that are part of _Performance Lab_ that can help with that, such as _Modern Image Formats_, which creates WebP versions of your images, _Image Placeholders_, which creates low-quality image placeholders (LQIP) for your images, and _Image Prioritizer_, which adjusts the order in which images are loaded. We can also use _Speculative Loading_, which is a feature that allows browsers to start loading resources before they are needed, which can help improve performance by reducing the time it takes for pages to load. Let's install the first three of those plugins and then run a new report to see what happens! Unfortunately, the _Modern Image Formats_ plugin says it creates the formats when images are uploaded, so it won't be able to help with the images that we have already uploaded, but it will help with any new images that we upload in the future.
16. You might find that when you re-run the report, a _Performance Lab_ plugin didn't actually help the situation. When that happens, what do you do? There's no option to disable the feature... Fortunately, when you activate one of the _Performance Lab_ plugins, you are installing and activating a separate plugin for that feature, so you can simply deactivate the plugin to disable the feature. For example, if you find that _Enhanced Responsive Images_ didn't help with the image delivery issue, you can simply go to _Plugins_, find _Enhanced Responsive Images_, and click _Deactivate_ to disable that feature. You can then run a new Lighthouse report to see how the score and recommendations change without that feature active. You can do this for each of the features to see which ones are actually helping with your site's performance and which ones aren't. This is a great way to test out different performance optimizations and see how they impact your site's performance in real time.
17. There are numerous other plugins for image optimization. One that I've had good experience with is _Smush_ by _WPMU DEV_, which can optimize your existing images as well as new images that you upload. You can install it and see if it helps, but that is not required.
18. There are also many plugins that can help with other performance issues, such as eliminating render-blocking resources and reducing initial server response time. _WPMU DEV_ has one called _Hummingbird_, _Automattic_ has _Jetpack Boost_, and there are a bunch of others. One of the oldest and most respected plugins in that space is called _W3 Total Cache_. You can try out any of these plugins to see if they help with the performance of your site, but again, that is not required.
